## THE MAIN UNRESOLVED BUG

User sees a blinking "typewriter" effect overlaid on the Win95 splash screen (the flag logo + progress bar). Despite many fixes, it persisted. The LAST fix applied (hiding desktop + CRT overlay during splash) was NOT tested by the user — they left frustrated.

**Root cause found via JS eval during splash:** `crtOverlay` (z-index 9000), `iconGrid` (z-index 200), and `win95Taskbar` (z-index 500) were ALL rendering on top of the splash (z-index 2001). The CRT scanline overlay at z:9000 was almost certainly the "blinking" effect.

**Fix applied:** `showWin95Splash()` now sets `document.getElementById('desktop').style.display = 'none'` and `document.getElementById('crtOverlay').style.display = 'none'`. Both are restored in `transitionToDesktop()`.

**FIRST ACTION: Hard-refresh (Ctrl+Shift+R) and check if blinking is gone.** If still there, remaining suspects:
1. CRT overlay might not be a child of `#desktop` — check if hiding desktop also hides it
2. Some other z-index element not caught by the eval
3. Ask user for phone video of what they see
