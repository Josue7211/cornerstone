  <action>
Read both SUMMARY files to understand what issues were reported in the Wave 1 and Wave 2 checkpoints. Fix each documented issue in order of severity.

**Common issues to check and fix (even if not reported):**

1. **GLTF model scale wrong** — If the cyberpunk-scene renders too large (model fills the whole viewport and you can't see anything) or too small (model is invisible), adjust:
   - `model.scale.set(0.05, 0.05, 0.05)` — start here
   - If too large: `model.scale.set(0.02, 0.02, 0.02)`
   - If too small: `model.scale.set(0.1, 0.1, 0.1)`
   - Also check: `model.position.y = -2` to lower the scene if it's floating

2. **GLTF model blocks portals** — If the scene geometry sits directly on top of the portal rings, raise portals:
   - In the PORTALS object, add a y-offset to portal light: `light.position.set(p.x, 5, p.z)`
   - Or lower the ground plane and GLTF together

3. **Bloom effect too strong** — If the screen is washed out white/overexposed:
   - Reduce bloom strength: `new UnrealBloomPass(..., 0.4, 0.4, 0.9)` (was 0.8, 0.4, 0.85)
   - Threshold 0.9 means only very bright objects bloom

4. **Chromatic aberration too strong** — If color fringing is distracting:
   - Reduce offset: `offset: { value: new THREE.Vector2(0.001, 0.001) }` (was 0.002)

5. **Portrait images 404** — If pioneer photos don't load:
   - Verify `src="./portraits/jensen.jpg"` paths in index.html
   - Check that `website/portraits/jensen.jpg` actually exists
   - If path is wrong, fix to `./portraits/jensen.jpg`

6. **GSAP not defined error** — If console shows "gsap is not defined" in main.js:
   - The GSAP script tag in index.html must be BEFORE the `<script type="module">` tag
   - In an ES module, `gsap` from a prior script tag is available as `window.gsap`
   - Fix: Change all `gsap.` calls in main.js to `window.gsap.` calls, OR ensure script order is correct

7. **Panel close buttons not working** — If clicking "BACK TO LOBBY" does nothing:
   - Verify button IDs in index.html match what's in main.js addEventListener calls
   - If IDs mismatch, either update index.html button IDs or update main.js to use correct IDs

8. **Mobile layout breaking viewport** — Per the spec, PC-only. But if viewport meta tag causes issues:
   - The `<meta name="viewport" content="width=device-width, initial-scale=1.0">` is correct and should stay
   - Do NOT add responsive breakpoints — this is intentionally PC-only

9. **Presentation slide grid overflow on narrow viewports** — If `.slide-implications` (2-column grid) breaks:
   - Add `@media (max-width: 700px) { .slide-implications { grid-template-columns: 1fr; } }`
   - Even though it's PC-only, browser DevTools may cause this in verification

10. **Loading bar never reaches 100%** — If GLTF onProgress total is 0 (server doesn't send Content-Length):
    - Fallback: show incremental progress using `progress.loaded` byte count as a visual indicator
    - `const pct = Math.min(99, Math.round(progress.loaded / 54000000 * 100));` (54MB estimated)
    - On success callback, immediately set to 100%: `loaderFill.style.width = '100%';`

After reading the SUMMARY files, address all documented issues. If no issues were documented, verify the above checklist anyway and fix anything found.
